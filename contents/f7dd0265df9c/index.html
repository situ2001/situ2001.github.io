JavaScript之What is this

发表于 2021-04-12 分类于 JavaScript
本文字数： 2.9k 阅读时长 ≈ 3 分钟

(浅解释,不深入)想必写js的，都会碰到关于this的问题吧？

this

A property of an execution context (global, function or eval) that, in non–strict mode, is always a reference to an object and in strict mode can be any value.

即是此时执行的context的一个属性，this的值表明了此时context是由谁执行的。

而context，可以理解为当前js代码运行时所在的环境吧？分为全局执行上下文和函数执行上下文，而每一个context，又都带有一个this属性。这里讲的this就是它了。

函数调用

在js里头，一般来说，函数调用方式有下面几种，前两个很常见，最后一个是Function.prototype.call

func(114514);
obj.func(114514);
func.call(obj, [114514]);

其中对于第三种的这个method，syntax如下

func.call([thisArg[, arg1, arg2, ...argN]])

thisArg在第一个位置，其实对于函数里头的this，MDN里头这样说

In most cases, the value of this is determined by how a function is called (runtime binding). It can’t be set by assignment during execution, and it may be different each time the function is called.

也就是说我们可以把this当成：某时某刻，这个函数是由谁调用执行的。this其实是当前执行该context的对象。在call()方法里头，函数里的thisArg就会是指定为传进去的this。

在nodejs下，上面的两种调用方式转化为call()。就会是如下代码所示（在浏览器里的话，第一个函数调用：非strict mode下就会是window，而strict mode下就是undefined）

func.call(undefined, [114514]);
func.call(obj, [114514]);

上面说完了，那，this真的就是一个对象吗？我们可以打印出来看看

const obj = {
    s: 2,
    i: 0,
    t: 0,
    u: 1
};

let func = function () {
    console.log(this);
};

func.call(obj); // { s: 2, i: 0, t: 0, u: 1 }

感觉说了一点啰嗦的东西…

大白话时间（不够水平）：在js里头，函数是一等公民，函数就是一个值，可以赋给其他变量。但函数声明在哪里（箭头函数除外），对于里面的this的值，不是最重要的。最重要的是：这个函数是谁调用的(或被绑定，下面说)。

于是乎，如果碰到有this的情况，我们可以做一点啰嗦的转化。

// foo is an object
foo.bar(114514);
// is equivalent to
foo.bar.call(foo, [114514]);

非常容易看到，此时的函数bar里面的this，就是foo。那么此时把里头的this都看成这个bar对象，不就EZ很多了吗？

this绑定

其实还有Function.prototype.bind这个方法，它是用于绑定this的，返回的是一个绑定了this的函数。注意，这个函数的第一个参数也是thisArg。

比如说

const obj = {
    s: 2,
    i: 0,
    t: 0,
    u: 1
};

let func = function () {
    console.log(Object.keys(this));
};

const foo = {
    dssq: 114514,
    fn: func.bind(obj) // a function that binds its this with obj.
};

foo.fn(); // [ 's', 'i', 't', 'u' ]

箭头函数

箭头函数没有属于自己的this，在箭头函数里头的this，都是从外面capture进来(就是你定义这个箭头函数的地方)的。比如

let test = function () {
    this.x = 114514;

    setTimeout(function () {
        console.log(this.x);
    }, 1000);
    
    setTimeout(() => {
        console.log(this.x);
    }, 1500);
};

let t = new test();
// undefined
// 114514

什么叫做箭头函数的this是在定义时候就已经被capture?看这个

const foo = function () {
    // Whatever `this` is here...
    var chopper = {
        name: 'Gogo',
        getName1: () => {
            return this.name;    // ...is what `this` is here.
        },
        getName2: function () {
            return this.name;
        }
    };

    console.log(chopper.getName1());
    console.log(chopper.getName2());
}

const obj = {
    name: 'Tony'
};

foo.call(obj);
// Tony
// Gogo

Constructor

对于construct pattern的函数，为什么里面必须要写类似this.x = 114这种语句才能让对象拥有这个property啊？为什么用new就能创建一个新对象啊？

其实理解this是什么之后，我们可以这样看待new operator，其实new运算符是分了几步的

比如我们这里有一个constructor

let foo = function () {
    this.x = 114;
    this.y = 514;
};

我们可以很方便地使用new foo()来得到一个新对象，但是流程能分成下面这些

let bar = Object.create(foo.prototype); // bar = {}
foo.apply(bar); // bar = { x: 114, y: 514 }

诺。也就是先把foo.prototype对象作为原型，创建一个新对象bar。然后以bar作为foo函数的this，调用foo函数。

// Now the property 'this' is bar;
this.x = 114; // is bar.x = 114
this.y = 514; // is bar.y = 514

此时的property就自然而然被加上咯。
